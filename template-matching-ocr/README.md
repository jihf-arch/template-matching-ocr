# 信用卡数字识别 (Credit Card Number OCR)

本项目是一个基于 OpenCV 和 Python 的光学字符识别（OCR）实战项目，主要用于识别信用卡上的数字。项目采用了**模板匹配 (Template Matching)** 的核心思想来实现数字的识别。

## 📁 文件结构

```text
template-matching-ocr/
│
├── images/                     # 测试用的信用卡图像文件夹
│   ├── credit_card_01.png
│   ├── credit_card_02.png
│   ├── credit_card_03.png
│   ├── credit_card_04.png
│   └── credit_card_05.png
│
├── ocr_a_reference.png         # OCR-A 字体参考模板图像（包含数字0-9）
├── ocr_template_match.py       # 主执行脚本：负责图像处理与数字识别
└── myutils.py                  # 自定义工具函数（轮廓排序、图像缩放等）
```

## 🚀 原理简介

本项目的数字识别流程主要分为两大部分：**模板处理** 和 **信用卡图像处理及匹配**。

1.  **模板提取：**
    *   读取 `ocr_a_reference.png` 图像，转换为灰度图和二值图。
    *   查找图像的外轮廓，按从左到右的顺序对轮廓进行排序。
    *   遍历每一个轮廓，提取出 0-9 每个数字的图像 ROI（感兴趣区域），并调整为统一大小（57x88），存入字典中作为模板。

2.  **信用卡号码识别：**
    *   **预处理：** 读取输入的信用卡图像，转换为灰度图，并执行**礼帽操作 (Top-hat)** 以突出明亮的数字区域。
    *   **边缘检测：** 使用 **Sobel 算子** 计算 X 方向的梯度，找出数字的垂直边缘。
    *   **形态学操作：** 通过**闭操作**（先膨胀再腐蚀）将相近的像素（通常是同一组数字）连接在一起，形成较大的块状区域。
    *   **二值化与轮廓查找：** 使用 OTSU 自动阈值进行二值化，并做额外的闭操作，之后查找所有轮廓。
    *   **过滤矩形区域：** 遍历所有轮廓并计算外接矩形，根据宽高比（长宽比通常在2.5到4.0之间）选取符合信用卡 4 个数字一组特征的候选区域。
    *   **字符分割与匹配：** 将选出的每一组数字区域按从左到右排序。在每个组内再次寻找单个数值的轮廓，并调整到与模板同样的大小。
    *   **模板匹配：** 使用 `cv2.matchTemplate` 找出每个裁剪出的数字区域与预先生成的 0-9 模板之间的最高匹配得分，并确定具体数字。最终结合各个数字及开头的数字判断信用卡的类型（如 Visa, MasterCard等）。

## 💻 使用方法

本项目通过命令行参数运行，必须提供两个参数：
*   `-i` / `--image` : 需要识别的信用卡图像路径
*   `-t` / `--template` : 模板图像的路径

**运行示例：**

打开终端或命令行，进入代码所在目录执行以下命令：

```bash
python ocr_template_match.py --image images/credit_card_01.png --template ocr_a_reference.png
```

运行过程中，脚本会通过 `cv_show` 自定义函数逐步展示预处理（如灰度图、二值化、轮廓绘制等）的过程。在弹出的图像窗口中按任意键（如 `Enter`）可进入下一步。最终会在原始图像上用红色框标注出识别的数字串，并在控制台打印出信用卡类型及卡号。

如果不需要逐步显示，可以根据需要在 `ocr_template_match.py` 中注释掉调用 `cv_show()` 函数的代码。
